## 一、重大发现：版本/架构不匹配

设计文档是为 **Forge 1.12.2 / Java 8 / `cn.openysm1122.*`** 编写的，但实际项目是 **Minecraft 1.21.x (26.1.2) / Fabric Loom + Architectury / Java 25 / `com.elfmcys.yesstevemodel.*` + `rip.ysm.*`**。

| 维度 | 设计文档假设 | 实际项目 |
|------|-------------|---------|
| MC 版本 | 1.12.2 Forge | 1.21.x (Fabric Loom + Architectury) |
| Java | 8 | 25 |
| GUI 基类 | `GuiScreen` / `GuiButton` | `Screen` / `Button` / `GuiGraphicsExtractor` |
| 配置系统 | 静态字段 | NeoForge `ModConfigSpec` |
| 包名 | `cn.openysm1122.*` | `com.elfmcys.yesstevemodel.*` |
| 构建工具 | ForgeGradle | Fabric Loom + Architectury |

**结论**：设计方案中没有一行代码可以直接复制使用，但功能意图完全可迁移，所有概念均有对等物。

## 二、总体可行性结论：✅ 完全可行

现有代码库已经为资源站功能准备好了基础设施。以下是逐项对照分析：

### 2.1 模型选择界面新增"资源站"按钮 — ✅ 可行

`PlayerModelScreen` [PlayerModelScreen.java](../common/src/main/java/com/elfmcys/yesstevemodel/client/gui/PlayerModelScreen.java:248) 已有完善的按钮布局系统。参考现有的"导入"按钮（`IconButton`，行 288-292），新增一个按钮仅需 ~10 行代码。

现有按钮组件：
- `FlatColorButton` — [FlatColorButton.java](../common/src/main/java/com/elfmcys/yesstevemodel/client/gui/button/FlatColorButton.java:1) 带文字的标准按钮
- `IconButton` — 图标按钮，支持 tooltip
- `ModIconButton` — 模组图标按钮

### 2.2 GuiResourceStation — ✅ 可行，有现成模板

`DownloadScreen.java` [DownloadScreen.java](../common/src/main/java/com/elfmcys/yesstevemodel/client/gui/DownloadScreen.java:1) 已是 "Coming Soooooooooooooooooooooooooon™" 占位符——这正是资源站的前身。

`ModelUploadScreen.java` [ModelUploadScreen.java](../common/src/main/java/com/elfmcys/yesstevemodel/client/gui/ModelUploadScreen.java:1) 提供了：
- 带进度条的完整参考实现
- 文件选择器
- 错误状态显示
- 后台任务与主线程回调

### 2.3 GitHub API 集成 — ✅ 可行（需新增 HTTP 客户端）

项目无现成的 HTTP 客户端。Java 25 可用 `java.net.http.HttpClient`（比设计文档的 `HttpURLConnection` 更现代）。GitHub Contents API 的 URL 转换逻辑可直接迁移。

### 2.4 模型下载到 `config/yes_steve_model/custom` — ✅ 可行，路径已存在

`ServerModelManager` [ServerModelManager.java](../common/src/main/java/com/elfmcys/yesstevemodel/model/ServerModelManager.java:83-97) 已定义：

```java
public static final Path FOLDER = Platform.getConfigFolder().resolve(YesSteveModel.MOD_ID);
public static final Path CUSTOM = FOLDER.resolve("custom");
```

即 `config/yes_steve_model/custom`。`ModelUploadScreen.openModelFolder()` 已在用此路径。

### 2.5 后台下载 + 主线程回调 — ✅ 可行

项目已有：
- `YSMThreadPool` — 通用线程池
- `modelPhraseExecutor`（`ThreadPoolExecutor`）— `ClientModelManager` [ClientModelManager.java](../common/src/main/java/com/elfmcys/yesstevemodel/client/ClientModelManager.java:296)
- `Minecraft.getInstance().execute(() -> {...})` — 主线程回调模式，遍布整个 `ClientModelManager`

### 2.6 文件名过滤 — ✅ 可行

`sanitizeFileName` 逻辑完全无版本依赖，直接迁移。

### 2.7 下载完成后刷新模型 — ⚠️ 需新增本地扫描方法

`ClientModelManager` 有 `onModelsLoaded` / `onModelsUpdated` 回调，`PlayerModelScreen` 实现 `IGuiWidget` [PlayerModelScreen.java](../common/src/main/java/com/elfmcys/yesstevemodel/client/gui/PlayerModelScreen.java:58) 接收通知。

但 **客户端本地新增文件不会自动触发模型加载**——当前模型加载完全依赖服务器同步协议。需新增：
- `ClientModelManager.scanLocalModels()` — 扫描 `CUSTOM` 目录并加载 `.ysm`/`.zip` 文件
- 或复用 `ServerModelManager.reloadPacks()` 的扫描逻辑

### 2.8 配置持久化 — ✅ 可行（需适配 ModConfigSpec）

项目使用 NeoForge `ModConfigSpec` [GeneralConfig.java](../common/src/main/java/com/elfmcys/yesstevemodel/config/GeneralConfig.java:1) 模式。设计文档的静态字段配置方式需要改写为：

```java
public static ModConfigSpec.ConfigValue<List<String>> REPO_URLS;
public static ModConfigSpec.IntValue SELECTED_REPO_INDEX;
public static ModConfigSpec.ConfigValue<String> IMPORT_DIR;
public static ModConfigSpec.BooleanValue AUTO_RELOAD_AFTER_DOWNLOAD;
```

## 四、风险与应对

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| **设计文档代码不能直接使用** | 🔴 高 | 全部用现代 Minecraft GUI API 重写，但逻辑和结构可迁移 |
| **本地下载后模型未加载** | 🟡 中 | 需新增 `ClientModelManager.scanLocalModels()` 方法 |
| **GitHub API 速率限制** | 🟡 中 | 第一版用 GitHub API + 后续支持 index.json 缓存 |
| **Java HTTP Client 在 Minecraft 环境兼容** | 🟢 低 | `java.net.http.HttpClient` 在 Java 25 可用；如遇类加载问题可退回 `HttpURLConnection` |
| **无 Mixin 约束** | 🟢 低 | 设计文档要求不用 Mixin，本项目 GUI 是纯扩展，确实不需要 |

```
第八步：新增 ClientModelManager.scanLocalModels()（本地模型扫描与加载）

```

## 六、后续扩展方向（设计文档提及但第一版可延后）

1. **index.json 格式支持**：加载更快，避免 GitHub API 速率限制
2. **多资源站 URL 管理**：切换/添加/删除资源站
3. **模型预览图**：在资源站列表中显示缩略图
4. **搜索/筛选**：按名称、标签、作者过滤
5. **文件夹递归下载**：支持 `.ysm`/`.zip` 之外的打包下载
6. **下载队列管理**：多文件并行/串行下载
